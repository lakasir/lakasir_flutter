# Update LakasirDatabase with New Schemas

## Context
- files: @lib/models/lakasir_database.dart
- Currently `LakasirDatabase.initialize()` only opens Isar with `[PrinterSchema, UnitSchema]`. All new offline models need their schemas registered in the Isar instance. Without this, the models won't be queryable.

## Goals
- Import all new offline model schema files into `lakasir_database.dart`
- Add all new schemas to the `Isar.open()` call: `OfflineUserSchema`, `OfflineCategorySchema`, `OfflineProductSchema`, `OfflineStockSchema`, `OfflineMemberSchema`, `OfflinePaymentMethodSchema`, `OfflinePendingTransactionSchema`, `OfflineCartSchema`, `SyncMetadataSchema`
- Ensure the app initializes without errors after schema changes
- Verify Isar can be opened with both old and new schemas (migration safety)

## Notes
- Isar schemas are additive — adding new schemas won't break existing `Printer` and `Unit` data
- The `Isar.open()` call with `name: 'isar'` must match the existing instance name
- This task must run AFTER all offline model `.g.dart` files have been generated by `build_runner`
- If Isar `getInstance("isar")` returns non-null (app was previously opened), the existing instance is reused — this is the existing pattern and must be preserved
- Test that `LakasirDatabase.initialize()` completes without throwing, and that `LakasirDatabase.isar` is accessible
