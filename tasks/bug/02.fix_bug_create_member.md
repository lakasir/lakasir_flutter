## Implementation

Fixed the issue where success feedback (snackbar) wasn't showing because it was called AFTER navigation.

### Root Cause
The snackbar was called AFTER `Get.back()` or `Get.until()`, so the navigation destroyed the context before the snackbar could be displayed.

### Changes

1. **lib/controllers/members/member_add_controller.dart**
   - Moved `Get.rawSnackbar()` BEFORE `Get.back()` so the snackbar displays before the screen is popped.

2. **lib/controllers/members/member_update_controller.dart**
   - `updateMember()`: Moved snackbar BEFORE `Get.until()`
   - `deleteMember()`: Moved snackbar BEFORE `Get.until()`

### Verification
- `flutter analyze lib/controllers/members/` passes with no issues
