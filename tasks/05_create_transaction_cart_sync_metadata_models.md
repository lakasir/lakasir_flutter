# Create PendingTransaction, Cart & SyncMetadata Models

## Context
- files: @lib/offline/models/pending_transaction_model.dart (new), @lib/offline/models/cart_model.dart (new), @lib/offline/models/sync_metadata_model.dart (new)
- Transactions are currently created via API calls (`PaymentSerivce.store`) with no offline queuing. When offline, transactions must be stored locally and queued for sync. Cart data is held in-memory via `CartController` and lost on restart. The `SyncMetadata` model tracks per-entity sync state for delta-sync.

## Goals
- Create `OfflinePendingTransaction` Isar collection with fields: `id` (autoIncrement), `payedMoney` (nullable double), `discountPrice` (double, default 0), `friendPrice` (nullable bool), `memberId` (nullable int), `tax` (nullable double), `note` (nullable String), `itemsJson` (String — JSON-encoded list of PaymentRequestItem), `isSynced` (bool, default false), `retryCount` (int, default 0), `createdAt`, `serverTransactionId` (nullable String), `errorMessage` (nullable String), `offlineReceiptNumber` (nullable String)
- Create `OfflineCart` Isar collection with fields: `id` (autoIncrement), `productId` (int), `quantity` (int), `price` (double — snapshot at add time), `discountPrice` (double, default 0), `addedAt` (DateTime). Use `@ignore` for a transient `OfflineProduct? product` field.
- Create `SyncMetadata` Isar collection with fields: `id` (int — entity name hashed as integer), `entityName` (String — e.g. 'products', 'categories'), `lastSyncAt` (nullable DateTime), `serverCount` (int, default 0), `localCount` (int, default 0), `syncStatus` (String, default 'idle')
- Include `part '*.g.dart'` and run `build_runner`

## Notes
- `itemsJson` stores the payment items as a JSON string because Isar doesn't support nested object lists natively. This will be deserialized when syncing to the server.
- `offlineReceiptNumber` uses the format `OFF-YYYYMMDD-XXXX` (e.g., `OFF-20260408-0001`) generated by `OfflineReceiptService`.
- `retryCount` maxes at 3 — after 3 failed sync attempts, the transaction is marked as permanently failed.
- `SyncMetadata.id` should be computed as `entityName.hashCode` to create a stable integer key from entity names like 'products', 'categories', etc.
- Cart persistence allows users to resume where they left off after app restart.
